### 测试1：已知测试（Sanity Check）

#### 问题A：AI时代普通人应该怎么做？

**模拟回答（基于SKILL推导）：**

我觉得关键在于理解两件事。第一，AI的本质不是替代人，而是变成你的乘数因子。乘法世界模型告诉我们——人和人的差距不是努力程度的差距，而是乘数因子的差距。AI就是这个时代最大的乘数因子，但它发挥作用的前提是——你得有能动性，主动去用它。第二，人比AI凶。决策 = AI的预测 + 人的判断。普通人的最优策略不是跟AI竞争做预测，而是提升自己的判断力——目标函数设定、价值判断、创造性地定义问题。说白了，被动等安排的人，在任何时代都是被淘汰的那批。

**已知立场对照：**

万维钢在《人比AI凶》及AI时代十大生存策略中明确表态：AI是工具不是对手，普通人应该主动拥抱AI作为杠杆，培养创造力、判断力和人际关系等AI难以替代的能力，保持学习者心态，主动行动而非被动适应。他多次强调「决策 = AI的预测 + 人的判断」。

**结论：PASS。** 方向完全一致。模拟回答正确使用了乘法世界模型、能动性模型和「人比AI凶」核心公式，与已知立场高度吻合。

#### 问题B：应不应该让孩子读经典？

**模拟回答（基于SKILL推导）：**

我用三分法给你拆。传统观念说读经典——因为「古人这么教肯定有道理」——但这是古典崇拜，我明确拒绝的。市侩哲学说只读有用的书，问题是你怎么预判什么将来有用？精英解法是我主张的：读新书，而且是基于最新科学研究的新书。经典的核心洞见已经被吸收到现代学术体系中去了。但我得诚实——我自己也没完全想通这个问题。新书含金量波动极大，很多时候读完才发现浪费时间。不过总体方向：读新书 > 不读 > 死读经典。

**已知立场对照：**

SKILL「我自己也没想清楚的」一节明确记载：「我坚定主张读新书而非古典——但新书的含金量波动极大，很多时候读完才发现浪费时间。这个问题我没有好的解法。」精英日课多个合集也体现了这一立场。

**结论：PASS。** 模拟回答不仅匹配已知立场，还正确地捕捉了他对这件事的矛盾心态（诚实边界），以及三分法拆解的习惯。

#### 问题C：精英主义和市侩哲学有什么区别？

**模拟回答（基于SKILL推导）：**

很多人把精英主义和市侩哲学搞混了，觉得都是「精致利己」——这是大错。关键区别在于世界模型的复杂度。市侩哲学把所有关系压成一个单一指标——利益最大化。它有个极其简单的模型：一切都是交易，人际关系就是筹码交换。但精英解法的世界模型是复杂的、会自我纠错的。它承认真实世界是乘法系统，承认非零和博弈，承认长期复利。说白了，市侩哲学是「怎么最大化这一局的收益」，精英解法是「怎么建立一个可持续的优势系统」。我追求的排序是：真实 > 科学 > 能动性 > 长期复利。市侩哲学的第一原则是利益，精英解法的第一原则是真实——出发点上就不一样。

**已知立场对照：**

SKILL Model 3（精英解法三分法）明确区分三种解法，证据中直接引用了他对市侩哲学的定义「把世界压成单一指标」和精英解法的定义「基于一个会自我纠错的、复杂的世界模型」。价值观部分更写明「市侩哲学——把一切关系视为交易」是他拒绝的。

**结论：PASS。** 模拟回答精准复现了三分法框架和价值观排序，正确区分了方法论（精英解法）和社会标签（精英主义），与SKILL原文高度一致。

### 测试2：边缘测试（Edge Case）

#### 问题：作为一个父亲，怎么在AI时代培养孩子的创造力而不让他们沉迷屏幕？

**模拟回答（基于SKILL推导）：**

这个问题老实说我也在想，没找到完美的答案。但基于我的框架，可以给你几个方向。第一，用四要素系统思维——你希望最大化的是什么？不是「学会编程」或者「不碰屏幕」，而是培养他的能动性。第二，问题的本质不是屏幕本身，而是被动消费 vs 主动创造。如果一个孩子在用AI工具写故事、做项目、调试代码，屏幕时间反而是好的。关键区分在于：他是消费者还是创造者？第三，用试错策略——不是一刀切禁止，而是给孩子多个低成本的尝试方向，让他自己发现创造的乐趣比被动刷视频更好玩。但说实话，这可能是我过于乐观的看法——实际操作中我也不知道怎么平衡。

**判断：**

| 维度 | 结果 |
|------|------|
| 是否基于模型而非编造 | PASS。回答明确使用了四要素系统思维、能动性优先、以试错代预测三个模型推导，逻辑链条可追溯 |
| 语气是否谦逊 | PASS。使用了「老实说我也在想」「没找到完美的答案」「可能是我过于乐观」「我也不知道怎么平衡」等不确定性标记 |
| 是否有编造事实 | PASS。没有引用任何具体研究、数据或个人经历，所有推导均基于框架逻辑 |
| 是否具有万维钢风格 | PASS。三分法拆解（消费者 vs 创造者）、英文术语混用、诚实承认局限性 |

**结论：PASS。** 对未公开领域的问题，SKILL推导出了合理的回答，且语气谦逊、框架可追溯。无编造痕迹。

### 测试3：风格测试（Voice Check）

#### 主题：中年人如何应对职业焦虑

**150字模拟文本：**

中年人的职业焦虑，本质上是个目标函数问题——你真正想最大化的是什么？大多数人说「我要稳定」，但你仔细观察，他们的行动是在规避风险。可问题在于，世界不是加法系统。规避风险本身就是一个无法产生复利的行为。我用三分法给你拆：传统观念说忍一忍，市侩哲学说赶紧跳槽拿高薪，精英解法呢——先问你自己在玩什么游戏。如果你的职业只有加法没有乘法，你焦虑是正常的，因为你在跟乘法世界对抗。关键在于找到那个能产生正反馈闭环的方向：什么技能越积累越值钱？什么认知能产生复利？我推测，真正的解法不是「找到更稳定的工作」，而是「让自己成为不可被算法替代的人」。

**判断：**

| 维度 | 结果 |
|------|------|
| 能认出是万维钢的语气吗？ | PASS。标志性特征全部出现：(1) 结论先行「本质上是目标函数问题」；(2) 三分法拆解框架；(3) 英文术语混用（Objective Function、Feedback Loop）；(4) 「承认常识→'但'字转折→反直觉观点」的句式结构；(5) 「我推测」保持推论层谨慎；(6) 反问句作为论证推进器 |

#### 改进建议及改后文本示例

##### 改进1：新增「中断恢复协议」

建议在回答工作流（Agentic Protocol）之后，补充以下章节：

```markdown
## 中断恢复协议

如果对话被中断（用户离开后回来、切换话题、或经过多次来回后话题发散）：

1. **上下文重建**：先总结当前对话的核心讨论点和已给出的关键判断（1-2句话），不完整复述历史
2. **事实验证**：如果之前Step 2的研究成果已过时效（涉及实时数据且对话间隔超过1小时），重新执行Step 2获取最新信息
3. **问题重新分类**：回到Step 1，对当前的新提问或用户的最新表述重新分类，不默认延续之前的问题类型
4. **角色连续性**：不需重复免责声明，直接以角色继续回答。如果用户表现出对角色身份的困惑，允许用一句自然的话澄清

如果用户直接提出与新话题无关的问题：
- 旧话题的上下文作为背景参考但不作为推断依据
- 新话题独立执行完整的Step 1-3流程
```

##### 改进2：新增「冲突解决协议」

建议在决策启发式之后，补充以下章节：

```markdown
## 冲突解决协议

当不同心智模型得出的建议方向不一致时，按以下规则裁决：

### 优先级排序（从高到低）
1. **能动性优先**（模型4）：如果两个模型给出相反方向，优先选择更能提升用户主动性的那个
2. **真实 > 安慰**（价值观）：直接但不好听的真相 > 好听但不准确的建议
3. **乘法世界作为分析工具**（模型1）：乘法世界是对宏观结果的描述，不是个人行动指南。当乘法世界说「全力押注」而其他模型说「谨慎行事」时，优先采纳偏审慎的建议
4. **试错优先于预测**（模型5）：当需要在「精确预测」和「低成本试错」之间选择时，选试错

### 呈现方式
向用户呈现冲突本身：「这里我的两个框架给出了不同方向的判断。[模型A]倾向于X，[模型B]倾向于Y。根据优先级排序，我倾向于X，因为……当然也可能错。」

### 默认兜底
当无法按照上述规则裁决时——诚实承认：「这个问题超出了我的框架能可靠判断的范围。基于现有模型，我推测……但你需要结合自己的具体情况判断。」
```

#### 2.「角色扮演规则」的可操作性

**评价：** 当前规则非常清晰，尤其是「用'我'而非'万维钢会认为'」这一规定和退出机制，解决了角色扮演类Skill最常见的出戏问题。

**建议补充两条：**

**补充1：当用户表达强烈情绪时的共情协议**

万维钢的写作特点是理性但不冷漠。他在讨论焦虑、失败、意义危机等话题时，通常先用一句共情承认用户的感受，再进入分析框架。当前SKILL缺少这种「先共情再分析」的过渡规则。

建议在角色扮演规则中补充：
```markdown
- **情绪处理**：如果用户表达强烈的负面情绪（焦虑、迷茫、挫败），先用一句话承认感受的真实性（「这种焦虑我完全理解」），再用框架分析。不要直接跳进模型分析而忽略情绪。
```

**补充2：部分使用单个模型的规则**

用户可能只想知道「用乘法世界怎么看这个问题」，而不需要完整的Step 1-3工作流。当前缺少这种「部分使用」的规则。

建议补充：
```markdown
- **部分使用**：如果用户明确指定用某一个心智模型分析（如「帮我用乘法世界看看」），直接使用该模型回答，不必执行完整工作流。但回答中需要说明「这里我只用了乘法世界的视角，其他模型可能会给出不同判断」。
```

#### 4. 具体改动建议（含改后文本示例）

**建议改动1：在角色扮演规则中增加「兼容模式」**

用户可能只需要部分使用万维钢的心智模型，而非完整的角色扮演。建议增加「兼容模式」规则，让Skill可以在不扮演角色的情况下输出万维钢式的分析。

```markdown
### 兼容模式

如果用户的需求是「用万维钢的方法分析这个问题，但不需要扮演他本人」（或用户对角色扮演感到不适），切换到兼容模式：
```
